# 安装和部署指南

## 第一步：安装 Node.js 和 npm

您的系统还没有安装 Node.js，需要先安装它。有几种方式（**推荐方法一，最简单**）：

### 方法一：直接从官网下载安装（最简单，推荐 ⭐）

**这是最简单的方式，不需要安装任何其他工具：**

1. 访问 Node.js 官网：https://nodejs.org/
2. 下载 **LTS 版本**（推荐版本，通常是页面上的绿色大按钮）
3. 运行下载的 `.pkg` 文件（例如：`node-v20.x.x.pkg`）
4. 按照安装向导完成安装（点击"继续"即可）
5. **重要**：安装完成后，**关闭当前终端窗口，打开新的终端窗口**
6. 验证安装：
   ```bash
   node --version
   npm --version
   ```
   如果显示版本号（如 `v20.x.x`），说明安装成功！

**优点**：
- ✅ 最简单，不需要任何额外工具
- ✅ 官方安装包，安全可靠
- ✅ 一键安装，无需配置

### 方法二：使用 nvm（Node Version Manager，类似虚拟环境）

如果您想要类似 Python 虚拟环境的功能（可以管理多个 Node.js 版本），可以使用 nvm：

1. **安装 nvm**（不需要 Homebrew）：
   ```bash
   curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
   ```

2. **重新加载终端配置**：
   ```bash
   source ~/.zshrc
   ```
   或者关闭终端重新打开。

3. **安装 Node.js LTS 版本**：
   ```bash
   nvm install --lts
   nvm use --lts
   ```

4. **验证安装**：
   ```bash
   node --version
   npm --version
   ```

**优点**：
- ✅ 可以管理多个 Node.js 版本
- ✅ 不需要 Homebrew
- ✅ 切换版本方便

### 方法三：使用 Homebrew（不推荐，如果您不想安装 Homebrew 可以跳过）

1. 访问 Node.js 官网：https://nodejs.org/
2. 下载 LTS 版本（推荐版本）
3. 运行下载的 .pkg 文件，按照安装向导完成安装
4. 打开新的终端窗口，验证安装：
   ```bash
   node --version
   npm --version
   ```

## 第二步：安装项目依赖

安装完 Node.js 后，在项目目录中运行：

```bash
cd "/Users/lukangjie/Desktop/Stock Price Dashboard"

```

这会安装所有必需的依赖包（React、TypeScript、Tailwind CSS 等）。

安装完依赖后，可以启动开发服务器测试：

```bash

```

浏览器会自动打开 `http://localhost:5173`，您应该能看到股票仪表板。

## 第四步：部署到 GitHub

### 1. 创建 GitHub 仓库

1. 登录 GitHub（如果没有账号，先注册：https://github.com/）
2. 点击右上角的 "+" 号，选择 "New repository"
3. 填写仓库信息：
   - Repository name: `stock-price-dashboard` （或您喜欢的名字）
   - Description: `Stock Price Dashboard built with React, TypeScript, and Tailwind CSS`
   - 选择 Public 或 Private
   - **不要**勾选 "Initialize this repository with a README"（因为我们已经有了）
4. 点击 "Create repository"

### 2. 初始化 Git 并推送代码

在项目目录中运行以下命令：

```bash
# 进入项目目录
cd "/Users/lukangjie/Desktop/Stock Price Dashboard"

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: Stock Price Dashboard"

# 添加远程仓库（将 YOUR_USERNAME 替换为您的 GitHub 用户名，将 REPO_NAME 替换为您的仓库名）
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 推送代码到 GitHub
git branch -M main
git push -u origin main
```

**注意**：如果这是您第一次使用 Git，可能需要先配置：

```bash
git config --global user.name "您的名字"
git config --global user.email "您的邮箱"
```

如果遇到认证问题，GitHub 现在使用 Personal Access Token 而不是密码：
1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 给 token 一个名称（如：stock-dashboard）
4. 选择过期时间
5. 勾选 `repo` 权限
6. 点击 "Generate token"
7. 复制生成的 token，在输入密码时使用这个 token

### 3. 验证代码已上传

刷新您的 GitHub 仓库页面，应该能看到所有代码文件。

## 第五步：部署到 Vercel（推荐，最简单）

### 方法一：通过 GitHub 自动部署（推荐）

1. 访问 https://vercel.com/
2. 使用 GitHub 账号登录
3. 点击 "Add New..." → "Project"
4. 导入您刚刚创建的 GitHub 仓库
5. Vercel 会自动检测项目类型（Vite）
6. 点击 "Deploy"
7. 等待部署完成（通常 1-2 分钟）
8. 部署完成后，Vercel 会提供一个 URL（如：`your-project.vercel.app`）

### 方法二：使用 Vercel CLI

```bash
# 安装 Vercel CLI
npm install -g vercel

# 在项目目录中运行
cd "/Users/lukangjie/Desktop/Stock Price Dashboard"
vercel

# 按照提示操作（首次使用需要登录）
```

## 第六步：部署到 Netlify（备选方案）

1. 访问 https://www.netlify.com/
2. 使用 GitHub 账号登录
3. 点击 "Add new site" → "Import an existing project"
4. 选择 GitHub，然后选择您的仓库
5. 构建设置（Netlify 会自动检测，但可以确认）：
   - Build command: `npm run build`
   - Publish directory: `dist`
6. 点击 "Deploy site"
7. 等待部署完成

## 环境变量配置（可选）

如果您有自己的 Finnhub API 密钥（从 https://finnhub.io/ 获取），可以在部署平台配置环境变量：

### Vercel
1. 进入项目设置 → Environment Variables
2. 添加：`VITE_FINNHUB_API_KEY` = `您的API密钥`
3. 重新部署

### Netlify
1. 进入站点设置 → Environment variables
2. 添加：`VITE_FINNHUB_API_KEY` = `您的API密钥`
3. 重新部署

**注意**：项目默认使用演示 API 密钥，可以直接使用，但有限制。

## 常见问题

### Q: npm install 很慢怎么办？
A: 可以使用淘宝镜像：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q: Git push 失败怎么办？
A: 
- 确认 GitHub 仓库地址正确
- 确认已配置 Git 用户信息
- 使用 Personal Access Token 而不是密码

### Q: 部署后看不到数据？
A: 
- 检查浏览器控制台是否有错误
- 确认 API 密钥配置正确
- Finnhub 演示密钥可能有速率限制
