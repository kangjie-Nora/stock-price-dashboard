# 关于本地开发服务器的说明

## 1. 什么是 localhost？

`localhost` 是一个特殊的域名，指向**您自己的电脑**。

- `http://localhost:5173/` = `http://127.0.0.1:5173/`（这两个地址完全一样）
- **只能在本机访问**，其他电脑无法访问
- 这是**本地开发服务器**的地址

## 2. 为什么需要启动服务器？

### 开发模式（npm run dev）

运行 `npm run dev` 时，Vite 会启动一个**开发服务器**，原因包括：

1. **热更新（Hot Reload）**：修改代码后浏览器自动刷新
2. **模块系统**：处理 TypeScript、JSX、CSS 等文件
3. **实时编译**：将现代 JavaScript/TypeScript 转换为浏览器可理解的代码
4. **开发工具**：提供错误提示、调试等功能

### 生产模式（npm run build）

运行 `npm run build` 后，代码会被编译成静态文件（HTML、CSS、JS），这时：
- 不需要开发服务器
- 可以直接用浏览器打开 `index.html`（但有些功能可能受限）
- 或者部署到 Web 服务器（如 Vercel、Netlify）

## 3. 别人的电脑能访问 localhost:5173 吗？

**不能！** `localhost` 只能在您的电脑上访问。

### 如果想让别人访问，有以下方法：

#### 方法一：部署到互联网（推荐）⭐

将代码部署到 Vercel、Netlify 等平台，这样全世界的人都能访问：

1. **推送到 GitHub**
2. **部署到 Vercel/Netlify**
3. **获得一个公开 URL**，例如：`https://your-app.vercel.app`

#### 方法二：局域网共享（仅限同一网络）

如果别人的电脑和您在同一个 WiFi 网络，可以：

1. 查找您的电脑 IP 地址：
   ```bash
   # Mac/Linux
   ifconfig | grep "inet "
   
   # 或者
   ipconfig getifaddr en0
   ```

2. 启动开发服务器时添加 `--host` 参数：
   ```bash
   npm run dev -- --host
   ```

3. 让别人访问：`http://YOUR_IP:5173`
   - 例如：`http://192.168.1.100:5173`

**注意**：这种方式只在同一 WiFi 网络下可用，且不安全，不建议用于生产环境。

#### 方法三：使用 ngrok（临时分享）

```bash
# 安装 ngrok（如果还没有）
# 访问 https://ngrok.com/ 下载

# 启动开发服务器
npm run dev

# 在另一个终端运行 ngrok
ngrok http 5173
```

这会生成一个临时 URL（如 `https://abc123.ngrok.io`），可以分享给别人，但：
- 免费版有使用限制
- URL 每次启动都会变化
- 仅用于临时测试

## 4. 用完一定要关掉服务器吗？

### 建议关掉，但不是必须的

**建议关掉的原因：**
1. **节省资源**：服务器会占用 CPU 和内存
2. **避免端口冲突**：端口 5173 被占用，下次可能启动失败
3. **节省电量**：笔记本电脑会消耗更多电量
4. **安全考虑**：开发服务器可能暴露代码和调试信息

**如何关掉：**
- 在运行 `npm run dev` 的终端窗口中按 `Ctrl + C`
- 或者直接关闭终端窗口

**不关掉的后果：**
- 通常不会有严重问题
- 但会一直占用系统资源
- 如果电脑重启，服务器会自动停止

## 5. 实际开发工作流

### 开发阶段
```bash
# 1. 启动开发服务器
npm run dev

# 2. 在浏览器访问 http://localhost:5173
# 3. 修改代码，浏览器自动刷新

# 4. 开发完成后，按 Ctrl+C 停止服务器
```

### 部署阶段
```bash
# 1. 构建生产版本
npm run build

# 2. 部署到 Vercel/Netlify/GitHub Pages
# 3. 获得公开 URL，所有人可以访问
```
