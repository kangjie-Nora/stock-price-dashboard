# 🚀 部署到 Vercel 详细步骤

## 前置准备

1. **确保代码已准备好**
   - ✅ 代码已完成并测试
   - ✅ 可以正常运行 `npm run dev`

2. **需要 GitHub 账号**
   - 如果没有，请先注册：https://github.com/

## 步骤一：初始化 Git 并提交代码

### 1. 检查 Git 状态

```bash
cd "/Users/lukangjie/Desktop/Stock Price Dashboard"
git status
```

如果显示 "not a git repository"，需要初始化：

### 2. 初始化 Git 仓库

```bash
# 初始化 Git
git init

# 配置 Git（如果还没配置过）
git config --global user.name "您的名字"
git config --global user.email "您的邮箱"
```

### 3. 添加并提交所有文件

```bash
# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: Stock Price Dashboard"
```

## 步骤二：创建 GitHub 仓库并推送

### 1. 在 GitHub 上创建新仓库

1. 访问 https://github.com/ 并登录
2. 点击右上角的 **"+"** 号 → **"New repository"**
3. 填写仓库信息：
   - **Repository name**: `stock-price-dashboard`（或您喜欢的名字）
   - **Description**: `Stock Price Dashboard built with React, TypeScript, and Tailwind CSS`
   - 选择 **Public**（公开）或 **Private**（私有）
   - **不要**勾选 "Initialize this repository with a README"（因为我们已经有了）
4. 点击 **"Create repository"**

### 2. 推送代码到 GitHub

在项目目录中运行：

```bash
# 添加远程仓库（将 YOUR_USERNAME 和 REPO_NAME 替换为您的实际信息）
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 重命名主分支为 main（如果还没有）
git branch -M main

# 推送代码到 GitHub
git push -u origin main
```

**如果遇到认证问题：**

GitHub 现在使用 Personal Access Token 而不是密码：

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 给 token 一个名称（如：`stock-dashboard`）
4. 选择过期时间（建议选择较长时间，如 90 天）
5. 勾选 `repo` 权限（这会自动勾选子权限）
6. 点击 **"Generate token"**
7. **重要**：立即复制生成的 token（只显示一次！）
8. 在输入密码时，使用这个 token 代替密码

### 3. 验证代码已上传

刷新您的 GitHub 仓库页面，应该能看到所有代码文件。

## 步骤三：部署到 Vercel（最简单）⭐

### 方法一：通过网页部署（推荐）

1. **访问 Vercel**
   - 打开 https://vercel.com/
   - 点击 **"Sign Up"** 或 **"Log In"**
   - 选择 **"Continue with GitHub"**（使用 GitHub 账号登录）

2. **导入项目**
   - 登录后，点击 **"Add New..."** → **"Project"**
   - 在 "Import Git Repository" 部分，您应该能看到您的 GitHub 仓库
   - 点击仓库旁边的 **"Import"**

3. **配置项目**
   - **Project Name**: 保持默认或修改（如：`stock-price-dashboard`）
   - **Framework Preset**: Vercel 会自动检测为 "Vite"
   - **Root Directory**: 保持 `./`（默认）
   - **Build Command**: `npm run build`（应该自动填充）
   - **Output Directory**: `dist`（应该自动填充）
   - **Install Command**: `npm install`（应该自动填充）

4. **环境变量**（如果需要）
   - 如果将来需要配置 API 密钥，可以在这里添加
   - 目前不需要，因为我们使用的是公开 API

5. **部署**
   - 点击 **"Deploy"**
   - 等待 1-2 分钟，Vercel 会自动：
     - 安装依赖
     - 构建项目
     - 部署到全球 CDN

6. **获得部署 URL**
   - 部署完成后，您会看到一个 URL，例如：
     - `https://stock-price-dashboard.vercel.app`
     - 或 `https://stock-price-dashboard-abc123.vercel.app`
   - 这个 URL 就是您的应用地址，**所有人可以访问**！

### 方法二：使用 Vercel CLI（可选）

如果您喜欢命令行：

```bash
# 安装 Vercel CLI
npm install -g vercel

# 在项目目录中运行
cd "/Users/lukangjie/Desktop/Stock Price Dashboard"
vercel

# 按照提示操作：
# - 登录 Vercel（首次使用）
# - 选择项目设置
# - 确认部署
```

## 常见问题

### Q: 部署失败怎么办？
A: 
- 检查 Vercel 部署日志中的错误信息
- 确保 `npm run build` 在本地能成功运行
- 检查是否有 TypeScript 错误

### Q: 如何自定义域名？
A: 
- 在 Vercel 项目设置中，点击 "Domains"
- 可以添加自定义域名（需要拥有域名）

### Q: 如何查看部署日志？
A: 
- 在 Vercel 项目页面，点击 "Deployments"
- 点击具体的部署，查看日志

### Q: 免费版有限制吗？
A: 
- Vercel 免费版对个人项目非常友好
- 有带宽和使用限制，但对于作业项目完全够用

## 部署到 Netlify（备选方案）

如果您想使用 Netlify 而不是 Vercel：

1. 访问 https://www.netlify.com/
2. 使用 GitHub 账号登录
3. 点击 **"Add new site"** → **"Import an existing project"**
4. 选择 GitHub，然后选择您的仓库
5. 构建设置：
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
6. 点击 **"Deploy site"**
